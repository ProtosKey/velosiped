# Velosiped System

> "Не такие метели в лицо летели" — когда Git кажется слишком тяжелым, а писать на Bash — западло.

**vls** — это хардкорная система контроля версий, написанная на чистом **C23** с использованием **POSIX** системных вызовов. Здесь нет места высокоуровневой магии, только `open`, `read`, `write` и голая арифметика битов.

## Технологический стек

*   **Язык:** C23 (только хардрок)
*   **Стандарт:** POSIX (никаких Windows-костылей, только POSIX)
*   **Криптография:** OpenSSL EVP API для MD5 хеширования
*   **Хранилище:** Самописное хранилище с адресацией по контенту
*   **Сборка:** CMake

## Как это устроено внутри

Вся твоя история живет в скрытой папке `.vls`:

*   `objects/` — Хранилище файлов. Именем файла является его MD5 хеш. Если контент одинаковый, в базе лежит только одна копия.
*   `commits/` — Архив слепков. Каждый файл здесь — это слепок состояния проекта в конкретный момент.
*   `stage/` — Опись проекта. Список того, что подготовлено к коммиту.
*   `head` — Указатель на текущую вершину истории.

## Команды

### Инициализация `init`

Инициализирует логово зверя. Создает структуру папок `.md` и файл `HEAD`.
```bash
vls init
```

### Отслеживание `add`

Берет твои файлы или целые папки, снимает с них хэширует их и крысит их копии в `objects/`, фиксируя пути в index.
```bash
vls init <files>
```

### Сохранение `commit`

Фиксирует текущее состояние проекта. Берет всё из `index/`, лепит метаданные и создает новую страницу в истории.
```bash 
vls commit <name>
```

### Возвращение `reset`

Откатывает состояние проекта в указанный момент.
```bash 
vls reset <hash>
```

### История `log`

Выводит всю историю вашего проекта.
```bash 
vls log
```

## Дисклеймер
